하얀바탕 검은글씨가 html

HTML(HyperText Markup Language)
-1990년 물리학자 Tim Berners-Lee가 정의
-표준화된 태그로 웹 페이지를 작성하는 언어

열리고 닫히고 = 태그 < >  = 뾰족괄호



요소 Element
시작태그 , 종료태그 
공 : 이름 같은데
차 : /의 유무. 종료태그는 /를 가지고 있음   "/로 문을 닫아주세요"

종료 태그가 없어도 되는 경우도 있지만, 종료 태그를 붙이는 습관을 들이는 것이 좋음.


태그에 추가 정보를 부여할 때 사용 = 속성
속성 = 태그에 추가적으로 설명을 추가적으로 달아주는 것 
속성은 작성하는 규칙이 있는데 
ex <img src="apple.png"/>
등호를 기준으로 왼쪽은 속성이름
등호를 기준으로 오른쪾은 속성 값

속성(attribute)은 여러개 달을 수 있어요.
왜? 속성 자체가 설명을 '추가적'으로 달아 주는 것이기에. 


태그와 속성은 대소문자 구분이 없음
ex. 
<HTML> ... </html>    시작태그 대문자, 종료태그 소문자 상관x
<img Src="heart.jpg">

☞보통 소문자로 통일

속성 값에 불필요한 공백 문자
-HTML5 표준에 어긋난다
-<img src="heart.jpg" width= "100">   = "100"앞에 공백한칸
이미지 가로가 100으로 맞춰져서 출력
-이미지 태그 = 화면에 이미지, 그림 보여주는 태그

속성 값은 반드시 따옴표("")로 묶어야 하나?
반드시 묶을 필요도 없고, 작은따옴표, 큰따옴표 상관 없음
ㄴ 안묶어도 괜찮지만 묶어주는게 암묵적인 룰

등호를 기준으로 오른쪽 값이 필요한데 
코딩이 만줄 되는데 가독성이 떨어지니까 

속성 값에 빈칸이 있는 경우 반드시 따옴표 사용 - 드물긴 하지만 구분이 필요할때는 빈칸 사용하고 이경우 따옴표 반드시 붙여야 한다

주석 : 개발자인 내가 참고하기 위해 쓰는 것 , 개발자를 위한 설명
